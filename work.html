Report on Blockchain Enables Voting

Abstract

Our country is a worlds largest democracy country and voting is a very crucial part of indian democracy. Indian politicians or any goverment is elected by only voting. The most common way of voting was in ancient time is Paper ballot system and not a days with EVM or electronic voting machine which is a software based voting. Because it is a software and control by an organization it is very difficult to maintain the security of votes. In E-voting was introduced to address those concerns. In this paper, we propose a blockchain enabled Voting system which is decentralized, secure with cryptography, non-alterable. By using this the voting system is transparent, secure, cost effective and immpossible to alter votes by anyone. For this we used a Ethereum Blockchain to store votes and smart contracts for voting. Our implementation was deployed on Ethereum blockchain to demonstrate usuability, scalability, efficiency.

Keyswords: Voting, Elections, Blockchain, Ethereum, Smart contracts

What is Blockchain ?

Before explaining what is blockchain, lets consider a scenario, There are two friends Alice and Bob, Alice wants to send some amount of money (10000$) to Bob. What he do, he will transfer money with his bank account and bank manager controls your transaction, he will deduct fix amount from your account and add to Bob's account then transaction complete. In this case bank and bank manager act as a center or third party on which we are depend. Not only money transaction but also in many sectors in which we will depend on some central party or authority. To establish trust between ourselves we depend on individual third party. For years we've depending on these middlemen to trust each other. Someone might ask “What is the problem depending on them?” The problem is they are single party or person or organization.

So how to overcome with above problem

In 2008 a person named Satoshi Nakamoto released a Paper “Bitcoin: A peer to peer Electronic Cash system”. It was the first application of Blockchain technology for cryptocurrency called Bitcoin. Continue with our above example, what if Alice wants to send money to Bob without any third party like bank. Blockchain will solve this problem. Blockchain is a growing list of transaction or records called blocks which are linked with cryptographic hash. Each block contains a cyrptographic hash of its previous block (just like linkedlist, contains address to next node), timestamp and transaction data and merkely root.


Key features of Blockchain

  • Peer to Peer network

  • Decentralized

  • Disdtributive Ledgers

  • Cryptographic hash

  • Proof of work


How Blockchain works:

Lets discuss how a blockchain work. There a network of Blockchain in which there are many nodes or CPU's which are connected Peer to peer network. When a new transaction is released it broadcast to all nodes in a blockchain. Every node have a ledger. Now to complete or verify the transaction every node need to finding a difficult proof of work which is a mathematical puzzle. When a node finds a proof of work or solve a problem successfully, it broadcast to all the nodes but the node who find the proof of work first will rewarded with some amount of Bitcoin (In case of Bitcoin) or any other cryptocurrency. When a transaction is verified it will give a unique transaction id and store in a particular block or written in a ledger. When this ledger is complete it will add in a Blockchain with previous hash of a block which is not alterable in future and Blockchain continously growing.

Security: Suppose some want to temper with data save in one block. But the hash of that block is saved in next block, so if data is altered according to hash property the hash of block is change so whole blockchain is disturbed. Secondly It will take 10 minutes to create a new block which include proof of work. If anyone want to alter the data in a block it is necessary to recalculate the proof of work and it will take much time and huge computational power, so it is immpossible to alter data in Blockchain.

Ethereum:

Initially when Bitcoin paper was released it was the only application of Blockchain. But after some times it is also possible to make applications on Blockchain for other sectors such as Finance, Real Estate, E-commerce. E-voting is one of them. Such type of applications which are build on Blockchain knows a DAPPS or Decentralized Application. But it was not possible to build DAPPS with bitcoin. So a new Blockchain was introduced known as Ethereum or the cryptocurrency for this is ether. By using Ethereum Blockchain building DAPPS are very efficient. For this we have to write some rules, now Smart contract comes into play. It is a kind contract in which some rules are specified for a particular task. For writing smart contracts there are many programming languages, Solidity is one of them and widely used. In Solidity we will write our smart contracts and deploy to Ethereum Blockchain and we get a unique address. Now when someone enter some input, it will in the form of transaction and a little amount of fee or GAS is required. .
