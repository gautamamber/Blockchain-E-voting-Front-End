Report on Blockchain Enables Voting

Abstract

Our country is a worlds largest democracy country and voting is a very crucial part of indian democracy. Indian politicians or any goverment is elected by only voting. The most common way of voting was in ancient time is Paper ballot system and not a days with EVM or electronic voting machine which is a software based voting. Because it is a software and control by an organization it is very difficult to maintain the security of votes. In E-voting was introduced to address those concerns. In this paper, we propose a blockchain enabled Voting system which is decentralized, secure with cryptography, non-alterable. By using this the voting system is transparent, secure, cost effective and immpossible to alter votes by anyone. For this we used a Ethereum Blockchain to store votes and smart contracts for voting. Our implementation was deployed on Ethereum blockchain to demonstrate usuability, scalability, efficiency.

Key features of Blockchain

  • Peer to Peer network: A peer to peer network is a group of computers or PC, which acts as a node for sharing data, files, content within a group. There is no central server in peer to peer network, each node or computer acts as a server as shown in Fig: 2. When a peer to peer network is established over the internet sharing of data is split between all nodes in that network.



  • Decentralized: Decentralized technology means doesn't rely on central authority or central point of control.



  • Disdtributive Ledgers: Shared and synchronized digital data


  • Cryptographic hash: Hashing in cryptography is which takes a message as an input and return a fixed size string of bytes (see Figure 4). The string is called Hash value. It is very difficult to calculate reverse the hash value. It is also not possible that two different messages has a same hash. Most commonly used hash functions are MD5, SHA256, SHA512



  • Public Key Cryptography: Public key or Asymmetric key cryptographic is an encryption scheme which uses public key and private key for message passing. Example: There are two person: A and B both have public key and private key. Public key is spread to all and private key is secret. If A wants to send message to B, He will encrypt the message with B's public key because B's public key is spread to all and not secure. Now B will get the encrypted message and decrypt that message with his Private key. So only B can see the message, example is shown in figure 5. In this Both can achieve end to end encryption of message.




How Blockchain works:

Lets discuss how a blockchain work. There a network of Blockchain in which there are many nodes or CPU's which are connected Peer to peer network. When a new transaction is released it broadcast to all nodes in a blockchain. Every node have a ledger. Now to complete or verify the transaction every node need to finding a difficult proof of work which is a mathematical puzzle. When a node finds a proof of work or solve a problem successfully, it broadcast to all the nodes but the node who find the proof of work first will rewarded with some amount of Bitcoin (In case of Bitcoin) or any other cryptocurrency. When a transaction is verified it will give a unique transaction id and store in a particular block or written in a ledger. When this ledger is complete it will add in a Blockchain with previous hash of a block which is not alterable in future and Blockchain continously growing.

Security: Suppose some want to temper with data save in one block. But the hash of that block is saved in next block, so if data is altered according to hash property the hash of block is change so whole blockchain is disturbed. Secondly It will take 10 minutes to create a new block which include proof of work. If anyone want to alter the data in a block it is necessary to recalculate the proof of work and it will take much time and huge computational power, so it is immpossible to alter data in Blockchain.
